# คำสั่ง OT สำหรับ Owner ผ่าน LINE

คำสั่งเหล่านี้รับเฉพาะ LINE User ID ที่ตั้งใน `LINE_OWNER_USER_ID` บัญชีอื่นจะถูกปฏิเสธ

> พนักงานไม่ต้องกดปุ่ม OT และจะไม่เห็นยอด OT หรือ Payroll ใน LINE

## สร้าง OT แบบเหมา

ใช้ชื่อพนักงานหรือ Employee ID ได้

```text
OT Win วันนี้ 200 16:00-18:00 เตรียมไส้เพิ่ม
```

```text
OT EMP001 2026-08-01 100 ช่วยปิดร้าน
```

รองรับวันที่:

- `วันนี้`
- `พรุ่งนี้`
- `YYYY-MM-DD`

ช่วงเวลา `HH:mm-HH:mm` ใส่หรือไม่ใส่ก็ได้ แต่ต้องมีเหตุผลและยอด OT

หลังส่งคำสั่ง:

1. ระบบสร้าง OT ID
2. ระบบไม่ส่งปุ่มหรือยอดเงินให้พนักงาน
3. Owner ได้เพียงข้อความยืนยันว่าบันทึกแล้ว
4. รายละเอียดอยู่ใน `HR_OT_REQUESTS`
5. ยอดยังไม่เข้า Payroll

## Owner ยืนยันหลังทำงาน

ต้องมีเวลา OUT ที่ถูกต้องก่อนอนุมัติยอด OT หากลืม OUT ต้องแก้ Attendance ก่อน

```text
OT อนุมัติ <OT_ID> 200 งานเสร็จครบ
```

หากไม่ใส่ยอด ระบบใช้ยอดเหมาเดิม:

```text
OT อนุมัติ <OT_ID>
```

ไม่อนุมัติ:

```text
OT ไม่อนุมัติ <OT_ID> พนักงานไม่ได้ทำงานเพิ่ม
```

เมื่อ Owner อนุมัติ ระบบจึงบวก OT เข้า Daily และ Weekly Payroll โดยไม่ส่งยอดให้พนักงานใน LINE

## ตัวอย่างครบหนึ่งรอบ

```text
Owner: OT Win วันนี้ 200 16:00-18:00 เตรียมไส้เพิ่ม
Employee: ทำงานและส่งรูป OUT ตาม Flow ปกติ
Owner: OT อนุมัติ ot_xxxxx 200 งานเสร็จครบ
Owner: ตรวจผลใน HR_OT_REQUESTS และ V52_PAYROLL
```

## ข้อความตอบ Owner

ระบบตอบเพียง:

```text
บันทึกรายการ OT เรียบร้อย ✅
OT ID: <OT_ID>
ดูรายละเอียดที่ชีท HR_OT_REQUESTS
```

ข้อความตอบจะไม่สรุปยอดค่าแรง ยอดหัก หรือยอด Payroll

## ข้อควรระวัง

- ห้ามส่ง Admin Token หรือ Secret ผ่าน LINE
- OT ID ไม่ใช่ Secret แต่ใช้สำหรับอ้างอิงรายการให้ถูกต้อง
- ยอด Final สามารถต่างจากยอด Preapproval ได้ แต่ต้องมี Audit Trail
- Fixed OT ควรถูกตรวจเทียบระยะเวลาจริงและข้อกำหนดกฎหมายแรงงานก่อนจ่าย
- Google Sheets เป็นหน้าตรวจรายละเอียด ส่วน D1 เป็น Source of Truth
